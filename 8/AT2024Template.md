# 1. Введение
Этот документ содержит спецификацию требований программного обеспечения (СТПО) для системы управления внутренними тренингами в IT-компании.

## Цели
Цель данной системы — оптимизировать процесс управления внутренними программами обучения в организации. Система обеспечивает эффективное планирование тренингов, назначение тренеров, управление участниками и отслеживание результатов. Этот документ описывает как функциональные, так и нефункциональные требования к системе, конструктивные ограничения и интерфейсы с другими системами.

## Границы применения
Система управления обучением предназначена для упрощения процессов планирования и мониторинга внутренних учебных мероприятий. Она предоставляет централизованную платформу, где руководители команд (Team Lead) могут заказывать тренинги, тренеры предлагать свои услуги, а сотрудники записываться на занятия. Система также отслеживает прогресс участников, хранит результаты обучения и генерирует отчеты для дальнейшей оценки эффективности и улучшения учебных программ.

## Термины и аббревиатуры
- **Team Lead**: Руководитель команды, который инициирует процесс заказа тренинга.
- **Тренинг**: Обучающее мероприятие, проводимое сотрудниками компании для других сотрудников.
- **N**: Максимальное количество участников, при достижении которого тренинг считается назначенным.

# 2. Общее описание

## Описание изделия
- **Интерфейсы системы**: веб-интерфейс для доступа к функциям системы через браузер и мобильное приложение для удобного доступа к системе на мобильных устройствах.
- **Интерфейсы пользователя**: удобное меню навигации для быстрого доступа ко всем функциям системы и интуитивно понятные формы для заполнения данных при регистрации на тренинги и отслеживании прогресса обучения.
- **Интерфейсы аппаратных средств ЭВМ**: персональные компьютеры с операционными системами Windows, MacOS, и Linux и мобильные устройства на базе iOS и Android.
- **Интерфейсы программного обеспечения**: взаимодействие с системами управления персоналом и корпоративными календарями через API для автоматической синхронизации данных и планирования уведомлений о сессиях обучения.
- **Интерфейсы коммуникаций**: электронная почта для отправки уведомлений о новых сессиях обучения и изменениях в расписании и внутренний чат для общения между участниками и тренерами.
- **Ограничения памяти**: оптимизированное использование оперативной памяти для обеспечения стабильной работы системы даже при высоких нагрузках, включая одновременное использование большого количества пользователей.
- **Действия**: пользователи могут регистрироваться на курсы, просматривать расписание, участвовать в учебных сессиях, проходить тестирование, получать сертификаты и отслеживать результаты обучения через отчеты.

## Характеристики пользователей
Основные пользователи — сотрудники компании, обладающие базовыми навыками работы с компьютером и способные пользоваться корпоративными системами. Система ориентирована на уверенных пользователей, но не требует глубоких технических знаний.
- **Team Lead**: Уверенные пользователи ПК, ответственные за планирование обучения в своей команде.
- **Тренеры**: Сотрудники компании с экспертными знаниями в определенной области.
- **Сотрудники**: Участники тренингов, которым требуется обучение.

## Ограничения
- Максимальное количество участников на тренинг ограничено значением N.
- Тренинги проводятся вне системы, результаты загружаются вручную.

## Предположения и зависимости
- Система предполагает интеграцию с корпоративной почтой и мессенджерами для уведомлений.
- Внесение результатов по тренингу происходит вручную после завершения мероприятия.
