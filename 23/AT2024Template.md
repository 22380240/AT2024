# 1. Введение
Данный документ спецификации требований к программному обеспечению (SRS) описывает функциональные и нефункциональные требования к системе, обеспечивающей сервис доставки непродовольственных товаров. Система позволит пользователям выбирать товары из наличия, оформлять заказы, осуществлять оплату и заказывать доставку с учетом специфических требований.

## Цели
Цель данного документа заключается в:
- Предоставлении четкого и структурированного описания функциональных и нефункциональных требований к системе доставки.
- Упрощении процесса разработки и тестирования программного обеспечения.
- Создании основы для дальнейшего анализа и уточнения требований от конечных пользователей.

## Границы применения
Этот документ распространяется на программное обеспечение, которое включает в себя:
- Интерфейс для пользователей, позволяющий выбирать товары из склада.
- Модуль обработки заказов, включая оплату и выбор параметров доставки.
- Систему управления доставками для курьеров, включающую возможность указания этажа и квартиры для массивных товаров.

## Термины и аббревиатуры
- Система — программное обеспечение, обеспечивающее сервис доставки непродовольственных товаров.
- Пользователь — лицо, использующее систему для выбора и заказа товаров.
- Курьер — лицо, ответственное за доставку товаров пользователю.
- Заказ — запрос пользователя на доставку определенного количества товаров с указанными параметрами.
- Этаж — номер этажа, на который необходимо доставить товар. 
- Квартира — номер квартиры, в которую требуется доставить товар.

## Краткий обзор
Данный документ структурирован следующим образом:
- Введение: Обзор документа, его цели, границы применения и термины.
- Функциональные требования: Описание функций, которые должна выполнять система.
- Нефункциональные требования: Описание производительности, безопасности и других качеств системы.
- Связанные документы: Упоминание других документов, которые могут быть полезны для понимания требований.
